# Nestify — AI агент для риелторов

## Суть проекта
Nestify — десктоп-приложение (веб-интерфейс через браузер) которое автоматизирует работу риелтора на Krisha.kz. Агент работает локально на компьютере пользователя, действует от имени его аккаунта на Krisha.kz, и использует Gemini API как мозг.

## Стек
- **Python** — основной язык
- **FastAPI** — бэкенд сервер
- **Playwright** — браузерная автоматизация (Krisha.kz)
- **Google Gemini API** — AI мозг агента
- **SQLite** — локальная база данных
- **HTML/CSS/JS** — фронтенд (минималистичный, стильный UI)

## Структура проекта
```
nestify/
├── main.py                  # точка входа — запускает FastAPI и открывает браузер
├── api/
│   ├── server.py            # FastAPI приложение
│   └── routes/

│   └── analyzer.py          # логика принятия решений агентом

├── ui/
│   ├── index.html           # SPA интерфейс
│   ├── styles.css           # стили
│   └── app.js               # логика фронтенда
├── assets/
│   └── logo.png

```

## Флоу запуска
```
Пользователь кликает на Nestify
        ↓
main.py запускает FastAPI на localhost:8000
        ↓
Автоматически открывается браузер
        ↓
Экран onboarding — ввод Gemini API токена
        ↓
Логин в Krisha.kz через Playwright
        ↓
Главный дашборд — задать параметры клиента
        ↓
Агент работает в фоне
```

## Флоу агента
```
Риелтор задаёт параметры клиента (район, бюджет, метраж, тип)
        ↓
Playwright открывает Krisha.kz от имени аккаунта риелтора
        ↓
Собирает новые объявления по фильтрам
        ↓
Gemini анализирует каждое (цена адекватна? описание норм? стоит писать?)
        ↓
По одобренным — агент пишет сообщение продавцу
        ↓
Все действия логируются в SQLite
        ↓
Риелтор видит результаты в дашборде
```

## UI/UX требования
- Минималистичный, стильный, современный дизайн
- Тёмная тема
- Никакого терминала для пользователя — всё через браузер
- Простой onboarding — 2 шага: токен → логин → готово
- Дашборд показывает: активные клиенты, найденные объявления, отправленные сообщения

## База данных (локальная SQLite)
Таблицы:
- `settings` — Gemini API токен, настройки
- `clients` — параметры поиска для каждого клиента риелтора
- `listings` — найденные объявления
- `messages` — история отправленных сообщений
- `actions_log` — лог всех действий агента

## Важные детали
- Агент действует **от имени аккаунта риелтора** — это легально (как CRM системы)
- Между действиями нужны **задержки** чтобы не выглядеть как бот
- Gemini токен хранится **локально** в SQLite, никуда не отправляется
- Сессия Krisha.kz сохраняется через Playwright cookies
- Поддержка нескольких клиентов одновременно

## Правила разработки
- Код пишем на Python 3.11+
- Async везде где возможно (FastAPI + Playwright async)
- Комментарии на русском языке
- Сначала делаем рабочий MVP, потом полируем
- Каждый модуль должен работать независимо

## Что НЕ делаем
- Не храним данные в облаке
- Не используем локальные LLM модели (нагружают компьютер)
- Не делаем массовый спам — агент действует осторожно и человекоподобно
- Не скрейпим данные в обход авторизации

## MVP порядок разработки
1. `main.py` + FastAPI сервер запускается и открывает браузер
2. Экран ввода Gemini токена + сохранение в SQLite
3. Логин в Krisha через Playwright
4. Базовый поиск объявлений по фильтрам
5. Анализ объявлений через Gemini
6. Отправка сообщений продавцам
7. Дашборд с результатами
